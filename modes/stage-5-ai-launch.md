# Stage 5: AI 产品上线前 — 合规最密集的阶段

> last_policy_review: 2026-04-15
> **本阶段是本 skill 最技术敏感、时间窗口最紧的部分。政策更新最快，上线前 30 天请再走一遍最新政策核对。**

## 何时进入

- 产品即将 go-live（国内 / 海外 / 两者）
- 或 已 live 但合规漏洞意识上升
- MVP 可用

## Disclaimer

> ⚠️ 本阶段同时涉及中国算法备案 / 大模型备案、GDPR、EU AI Act、PIPL 出境、版权、深度合成标识等。
> **任何一项单独都值一家专业律所。Skill 只给地图，每项落地必须找律师 + 合规咨询。**

## 核心分流

```
面向中国公众？
├─ 是 → 国内三轨（算法备案 / 大模型备案 / 登记）必走一轨或多轨
└─ 否 → 跳过国内三轨

面向欧盟用户？
├─ 是 → GDPR + Art. 27 + EU AI Act Art. 50（2026-08 起） + GPAI（自研）
└─ 否 → 跳过

面向加州用户？
├─ 是 → CCPA/CPRA + SB-53（frontier）+ Colorado AI Act（若涉 consequential decision）
└─ 否 → 跳过

训练语料含中国 PI？
├─ 是 → PIPL 出境按阈值
└─ 否 → 跳过
```

## 决策 1：国内三轨走哪一条（或多条）

### 决策树

```
自研模型？
├─ 是 → 面向公众？
│       ├─ 是 → 生成内容？
│       │       ├─ 是 → 走 [B] 大模型备案 + [A] 生成合成算法备案（双备）
│       │       └─ 否 → 走 [A] 算法备案
│       └─ 否（纯 ToB / 内部）→ 条件免
└─ 否（调 API）→ 面向公众？
        ├─ 是 → 走 [C] 大模型登记
        └─ 否 → 可能免
```

> 详细材料 / 流程 / 成本见 `appendix/ai-filing-guide.md`

### 时间规划
- [C] 登记 → 上线前 **2 个月**
- [A] 算法备案 → 上线前 **3 个月**
- [B] 大模型备案 → 上线前 **6 个月**
- [A] + [B] 双备 → 上线前 **6-9 个月**

### 服务商
- 综合顾问：锦天城 / 中伦 / 观韬中茂 / 安杰世泽
- 内容安全一体化：数美 / 易盾 / 阿里云盾 / 腾讯云天御
- 第三方安评机构：中国信通院 / 电子标准院

## 决策 2：语料合规（B 轨核心卡点）

### 2026 最新要求

| 维度 | 要求 |
|---|---|
| **境外语料占比** | **≤30%** — 2026 起严查 |
| **授权证明** | 每个语料源（公开抓取 / 购买 / UGC）均须授权链条可追溯 |
| **违法不良信息过滤** | 训练前 + 训练中 + 训练后三道过滤 |
| **个人信息** | 无授权不得含 PI；未成年人信息强化保护 |
| **版权** | 不得爬取违规盗版站；尊重 opt-out 机制（呼应 EU AI Act 版权政策） |

### 实操

- **数据溯源台账**：每批数据 source / license / 日期 / 处理人
- **授权书模板**：公开数据用 Dataset License（CC / MIT / Apache）+ 商业数据用 DPA / License
- **境外占比测算**：按 token / 文件数 / 字节数三种统计方式自备（监管抽查任一）
- **脱敏 + 去标识化**：PI 处理流水

### 红旗

- 🚩 境外语料 >30% → 大模型备案打回
- 🚩 语料源无授权书 → 一律打回
- 🚩 爬取 LibGen / SciHub / 盗版小说站 → 爆炸红旗

## 决策 3：内容标识（2026 新增硬要求）

### 深度合成 + 生成式内容强制标识

- **显式水印**：肉眼可见的 "AI 生成 / AIGC / 深度合成" 标识（角标 / 底部文字 / 语音开场白）
- **隐式哈希 / 元数据**：机器可读，嵌入文件元数据或内容 hash 入公共登记
- **两者缺一不可**（2026 CAC 抽查要求）

### 对应 EU AI Act Art. 50（2026-08 起）

- AI 生成 synthetic 内容必须机器可读标识
- 与中国双水印要求在技术路线上 **大致兼容**（但法律文本要求不完全一致，需并行披露）

### 工程实现

- 图像：隐式用 **C2PA / SynthID** 系列 + 显式加视觉标识
- 视频：C2PA + 视频片头 / 角标
- 语音：含水印算法（OpenAI / ElevenLabs 已有方案）
- 文本：显式标注更重要（元数据方案实际效果弱）

## 决策 4：GDPR 最小清单（若欧盟用户 ≥1）

- [ ] Privacy Policy（包含：controller / 处理目的 / 法律基础 / 保留期 / 用户权利 / 第三国传输）
- [ ] Cookie Banner（非必需 cookie opt-in）
- [ ] DPA（与所有数据处理方签）
- [ ] SCC（数据出欧盟必签，China/US 传输）
- [ ] **Art. 27 欧盟代表**（Prighter / VeraSafe / EDPO，€200-400/月）
- [ ] DPO（高敏感 / 大规模才需）
- [ ] 用户权利响应机制（30 天内）
- [ ] **DPIA**（AI 产品高风险处理几乎必做）

**罚款：** 最高 €20M 或全球营收 4%。

## 决策 5：EU AI Act — GPAI Provider 义务（2025-08-02 起，若自研）

### 是否触发

- 自研基础模型 / 通用能力模型 → 是
- 仅调用上游 API → 否（但上游须合规）

### 义务

1. **技术文档**（Annex XI）
2. **训练数据摘要**（AI Office 公共模板）
3. **版权政策**（opt-out 尊重 + 不爬违禁）
4. **下游信息披露**（帮助下游完成其义务）
5. **若 systemic risk**（≥ 10^25 FLOPs 训练算力）— 模型评估 + 对抗测试 + 严重事件上报 + 网络安全

### Code of Practice 路径

签 Code of Practice → **推定合规**，直到欧洲标准落地。成本比自证合规低。

### 中国训练模型 服务欧盟的冲突点

- **PIPL 要求：不披露涉重要数据 / 核心数据细节**
- **AI Act 要求：公开训练数据摘要**

**建议 split strategy：**

| 披露层 | 国内 | EU |
|---|---|---|
| 具体数据集明细 | 备案内部披露 | **不披露**（法域冲突 → 按 PIPL 底线执行） |
| 语料类别 + 粗占比 | 备案内部 | 公开摘要（AI Office 模板允许抽象） |
| 版权政策 | 必有 | 必有（可复用文本） |
| 授权链条 | 内部记录 | 可用结构性声明（"来源于合作伙伴、经授权的公开数据、自有标注"）代替逐个披露 |

**建议：律师并行核对 PIPL + AI Act 披露边界，不要自己编公式。**

## 决策 6：PIPL 出境（若训练 / 用户数据含中国 PI）

### 三档阈值（2024-03 CAC 新规）

| PI 量级（累计 / 自每年 1 月 1 日） | 动作 |
|---|---|
| <10 万人（非敏感）& 无敏感 PI | 免申报 |
| 10-100 万人（非敏感）或 <1 万人敏感 | SCC 备案 或 认证（二选一） |
| ≥100 万人（非敏感）或 ≥1 万人敏感 | 安全评估（省级 → 国家 CAC） |
| CIIO 或 涉"重要数据" | 一律安全评估 |

### 操作

- **训练数据出境场景**：模型在国内训练但用户行为日志回传海外研发团队 → 可能踩出境
- **用户数据场景**：产品面向国内用户但海外团队访问日志 → 出境
- **推荐路径（早期）**：数据架构物理隔离（国内数据不出境 / 模型权重出境但去标识化）→ 免申报

### 红旗

- 🚩 月活 >100 万但没做安评 → 千万级罚款 + 暂停
- 🚩 训练语料含中国用户行为但境外训练 + 无授权 → 涉重要数据嫌疑

详见 `appendix/compliance-matrix.md` § 二。

## 决策 7：US 州法合规（若加州 / 科罗拉多用户）

### CCPA / CPRA

触发（任一）：
- 年营收 ≥$25M
- 年处理 ≥100k CA 居民 PI
- ≥50% 营收来自卖/分享 PI

必做：
- Privacy Notice at collection
- "Do Not Sell or Share My Personal Information" 首页 footer 链接
- Sensitive PI 使用限制告知

### California SB-53（TFAIA，2026-01-01 已生效）

触发：
- frontier 大模型方（大参数量 / 大算力）
- 面向 CA 公众

义务：
- 公示 **Frontier AI Framework**（技术 + 组织措施防 catastrophic risk）
- 年度更新 + 重大变更 30 天内公告
- 严重事件上报

### Colorado AI Act（SB24-205，延至 2026-06-30 生效）

触发：
- 部署"consequential decision" AI（影响教育 / 就业 / 金融 / 医疗 / 住房 / 保险 / 法律服务 / 政府 benefits）
- 影响科罗拉多居民

义务：
- Impact Assessment
- Risk Management Program
- 用户通知 + 人工复核机会

> 2026-Q1 科罗拉多立法方提 Proposed ADMT 框架，可能替代原 Act 并推至 2027-01-01。**上线前再次复核最新进展。**

## 决策 8：软著 2026 新规（若提软著）

### 2026 版核心变化

1. **核心功能描述字数从 200 提到 500-1300 字**
2. **申请人需实名承诺：未使用 AI 开发 / 写代码 / 生成材料**
3. **申请表签字页需手写"未使用 AI 工具"承诺**
4. **违规使用 AI → 个人 / 企业信用黑名单 + 国家信用联合惩戒**
5. 若确含 AI 生成内容：需附 "AI 生成内容声明"（工具 / 版本 / 用途 / 人工修改）

### 实操

- 软著继续有价值（补贴 / 资质 / 招标 / 融资 DD）
- 但不要用 AI 批量生成代码 / 文档申请软著
- 真人手写申请文本 + 手写承诺

### 红旗

- 🚩 让代理公司批量 AI 写软著申请 → 征信污染
- 🚩 软著材料含 AI 生成但未声明 → 撤销 + 黑名单

## 决策 9：模型 License 底线（Red Flag 4）

### Llama Community License 风险点
- **月活 7 亿上限**
- **不得用 Llama 输出改进其他 LLM**

### 替代
- Qwen / DeepSeek / Mistral（Apache 2.0 部分版本）
- 自训练
- 闭源商用 API

### AGPL 禁区（Red Flag 5）

- SaaS 模式禁用
- 典型陷阱：MongoDB（旧版） / 部分向量数据库 AGPL 版
- 替代：Postgres / pgvector

## 典型避坑场景

**场景 1：ByteDance Doubao 某版本下架（2024）**

2024 年某时期豆包某模型版本因语料 / 内容审核瑕疵被要求暂停对外服务并整改。**启发：大厂都会被打回，小厂更要提前 6 个月跑备案不要赌 3 个月档期。**

**场景 2：某文生图公司境外语料 60% 备案被打回**

某垂直文生图 AI 公司 2025 年底提大模型备案，第三方安评报告显示境外 LAION 系数据集占 60%。打回 → 重新采购国内授权数据集 → 延期 4 个月上线。

**场景 3：某数字人公司无深度合成双水印被 CAC 通报**

某 2B 数字人 SaaS 只做了显式角标未做隐式水印，2026-Q1 被抽查通报 → 48 小时整改 + 客户批量退款。

## 必查清单

### 国内上线
- [ ] 算法备案 / 大模型备案 / 登记（至少一轨）
- [ ] 语料合规（境外 ≤30% / 授权链 / 过滤）
- [ ] 双水印（显式 + 隐式）
- [ ] 深度合成独立备案（如涉人脸/语音/数字人）
- [ ] 用户协议 + 隐私政策
- [ ] 内容安全机制（关键词库 + 模型 + 人工）
- [ ] 投诉举报渠道
- [ ] 等保 2.0 定级 + 备案

### 欧盟上线
- [ ] Privacy Policy + Cookie Banner + DPA + SCC
- [ ] Art. 27 欧盟代表合同
- [ ] EU AI Act Art. 50 标识机制（2026-08 前到位）
- [ ] GPAI 义务（自研） — 技术文档 + 训练数据摘要 + 版权政策
- [ ] Code of Practice 签署评估

### 加州 / 科州
- [ ] CCPA/CPRA Notice + "Do Not Sell" 链接
- [ ] SB-53 Frontier AI Framework（若触发）
- [ ] Colorado AI Act Impact Assessment（若触发）

### PIPL 出境
- [ ] 数据架构图（哪些数据出 / 哪些不出）
- [ ] 阈值测算
- [ ] SCC / 安评 / 认证路径选择 + 落地

### License
- [ ] 基模 License 核对（Llama 条款 / Apache 2.0 等）
- [ ] 依赖树 AGPL 扫描
- [ ] Prior IP 白名单
