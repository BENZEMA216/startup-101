# References: Ideas & Validation — 想法与验证

## Tier 1

### Paul Graham — How to Get Startup Ideas

- **作者/来源：** Paul Graham (YC 联创)
- **发布：** 2012-11
- **URL：** https://www.paulgraham.com/startupideas.html
- **适用阶段：** Stage 0 / Pre-Stage 1
- **一句话价值：** 创业想法不是"想"出来的，是"注意"出来的 — 寻找自身 organic problem
- **核心要点：**
  1. 好想法在早期看起来像坏想法
  2. "有机"想法（你自己就有这个痛点）优于刻意想
  3. 寻找具体而非抽象的用户
  4. "Live in the future, build what's missing"
- **AI 创业者特别启发：** 不要把"用 AI 做 X"当成想法起点 — 想法起点应该是 X 这件事为什么做得不好

### Paul Graham — Do Things That Don't Scale

- **作者/来源：** Paul Graham
- **发布：** 2013-07
- **URL：** https://www.paulgraham.com/ds.html
- **一句话价值：** 早期创业的核心动作是 manual / 不 scale 的 — 一个个拉用户、一条条聊需求
- **核心要点：**
  1. Airbnb 亲自拍房源照片 / 创始人飞 NYC
  2. 早期用户是"找来"不是"增长来"
  3. 做"Concierge MVP" — 先人力后自动化
  4. 重型制造 / 硬件初期也能 manual

### Paul Graham — Before the Startup

- **作者/来源：** Paul Graham
- **发布：** 2014-10
- **URL：** https://www.paulgraham.com/before.html
- **适用阶段：** Stage 0
- **一句话价值：** 创业最大的风险是创始人本身是否"准备好" — 不是技能问题，是取舍问题
- **核心要点：**
  1. 创业对比上班的不对称痛苦
  2. 联合创始人的选择比技术更重要
  3. 不要为"创业"而创业，为真问题创业

### 陆奇 — 我的大模型世界观 / AI 创业演讲

- **作者/来源：** 陆奇（奇绩创坛创始人）
- **发布：** 2023-2025 多场演讲汇编
- **URL：** 散见 B 站 / 公众号「奇绩创坛」/ 晚点 LatePost
- **典型链接：** https://mp.weixin.qq.com/s/_ZvyxRpgIA4L4pqfcQtPTQ（2023-04 复旦演讲）
- **一句话价值：** 中文世界最系统的"AI 范式转移"创业者导读
- **核心要点：**
  1. 大模型是"信息文明 → 知识文明"的基础设施
  2. "模型为先、场景为王、生态共赢"
  3. AI 创业者应分清 model 层 / infra 层 / app 层的机会
  4. 不要跟大厂硬拼通用能力，做场景化壁垒

### Sam Altman — Startup Playbook

- **作者/来源：** Sam Altman (前 YC President, OpenAI CEO)
- **发布：** 2015（常看常新）
- **URL：** https://playbook.samaltman.com/
- **一句话价值：** 一页纸创业手册 — 从 idea 到 team 到 execution
- **核心要点：**
  1. "The most common mistake founders make is to solve problems no one has."
  2. 团队匹配 > 团队精英
  3. 先 make something people want，再谈所有其他
  4. 创业成功三要素：idea + product + team + execution
